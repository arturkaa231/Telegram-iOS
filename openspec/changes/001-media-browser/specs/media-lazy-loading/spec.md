# Capability: media-lazy-loading

**Status:** new
**Owner:** Multigram iOS app
**Persona:** end_user

## Description

Ленивая подгрузка медиафайлов чата с бесконечной прокруткой. Загружает данные порциями через Telegram shared media API, подгружая следующую порцию при приближении к концу списка.

---

## Functional Requirements

### Requirement: Первоначальная загрузка порции

При открытии медиабраузера SHALL загружаться первая порция медиафайлов (20 элементов).

#### Scenario: Первая порция загружается

```
GIVEN пользователь открыл медиабраузер в чате с 50+ медиафайлами
WHEN модальное окно отобразилось
THEN загружаются первые 20 медиафайлов
  AND файлы отображаются в списке (новые сверху)
  AND индикатор загрузки скрывается
```

#### Scenario: Индикатор загрузки при первой порции

```
GIVEN пользователь нажал кнопку медиабраузера
WHEN данные загружаются с сервера
THEN в области списка отображается индикатор загрузки (spinner)
```

### Requirement: Подгрузка следующей порции при скролле

Следующая порция SHALL загружаться автоматически при приближении к концу текущего списка.

#### Scenario: Бесконечная прокрутка

```
GIVEN в списке отображаются 20 медиафайлов
  AND в чате есть ещё файлы
WHEN пользователь проскроллил до 15-го элемента (75% списка)
THEN автоматически начинается загрузка следующих 20 файлов
  AND в конце списка отображается индикатор подгрузки
  AND после загрузки новые файлы добавляются в конец списка
```

#### Scenario: Конец списка

```
GIVEN все медиафайлы чата загружены
WHEN пользователь проскроллил до конца
THEN индикатор подгрузки НЕ отображается
  AND дополнительных запросов к API НЕ происходит
```

### Requirement: Фильтрация по табам

При переключении таба SHALL загружаться новая порция с соответствующим фильтром.

#### Scenario: Смена таба сбрасывает список

```
GIVEN активен таб "Все файлы", загружено 40 элементов
WHEN пользователь переключает на таб "Закреп"
THEN текущий список очищается
  AND загружается первая порция закреплённых файлов (20 элементов)
  AND отображается индикатор загрузки на время запроса
```

### Requirement: Повтор при ошибке

При ошибке сети SHALL предоставляться возможность повторной загрузки.

#### Scenario: Ошибка сети при подгрузке

```
GIVEN загружена первая порция файлов
  AND пользователь скроллит к концу списка
WHEN загрузка следующей порции завершается с ошибкой сети
THEN в конце списка отображается сообщение "Ошибка загрузки"
  AND отображается кнопка "Повторить"
  AND ранее загруженные файлы остаются в списке
```

#### Scenario: Успешный повтор

```
GIVEN отображается ошибка загрузки с кнопкой "Повторить"
WHEN пользователь нажимает "Повторить"
THEN начинается повторная загрузка следующей порции
  AND отображается индикатор загрузки
```

---

## Non-Functional Requirements

- **Первая порция:** загрузка SHALL завершаться < 1000ms (p99) при стабильном соединении
- **Размер порции:** 20 элементов на запрос
- **Порог подгрузки:** триггер на 75% скролла текущего списка
- **Кэширование:** загруженные элементы SHALL сохраняться в памяти до закрытия модалки
- **Дедупликация:** при повторной подгрузке SHALL НЕ дублировать уже загруженные элементы

## Service Boundaries

- **Владелец:** Multigram iOS app, модуль MediaBrowser
- **Runtime:** Swift (iOS)
- **Зависимости:**
  - TelegramCore `searchMessages` API (hard) — источник данных
  - media-browser-modal (hard) — UI для отображения

## API Sketch

Используется существующий Telegram API через TelegramEngine:

```
// Получение shared media чата
TelegramEngine.Messages.searchMessages(
    peerId: EnginePeer.Id,
    query: "",
    filter: MessageTags,       // .photo, .video, .file, .music, .pinned
    limit: 20,
    offsetMessageId: EngineMessage.Id?  // для пагинации
) -> Signal<[EngineMessage], NoError>
```

Новый API не требуется — используются существующие методы TelegramEngine.

## Data Model

**LoadingState:**

| Значение | Описание |
|----------|----------|
| idle | Нет активной загрузки |
| loading | Идёт загрузка порции |
| error(String) | Ошибка загрузки |
| exhausted | Все элементы загружены |

## Error Conditions

| Код | Описание | Поведение |
|-----|----------|-----------|
| NETWORK_ERROR | Таймаут или ошибка сети | Показать ошибку + кнопка "Повторить", сохранить загруженные данные |
| EMPTY_RESPONSE | API вернул 0 элементов | Перейти в состояние exhausted, скрыть индикатор |
